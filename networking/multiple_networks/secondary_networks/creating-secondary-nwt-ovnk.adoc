= Creating secondary networks on OVN-Kubernetes
include::_attributes/common-attributes.adoc[]
:context: configuring-additional-network-ovnk

toc::[]

As a cluster administrator, you can configure an additional secondary network for your cluster using either the `UserDefinedNetwork` (UDN) resource or the `NetworkAttachmentDefinition` (NAD) resource. When creating additional networks, you might consider the flexibility and advantages UDN provides over NAD.
